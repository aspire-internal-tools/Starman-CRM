# Aspire Brand Standard For Starman

Last updated: 2026-07-14

Source: `branding/aspire-branding-skill` supplied from the Aspire brand guideline reference.

## Required Brand Tokens

| Token | Value | Use |
|---|---|---|
| Aspire dark green | `#10574C` | Header, sidebar, dark chrome, white-logo background |
| Aspire web green | `#157566` | Primary buttons, links, active states |
| Aspire light green | `#308D7F` | Hover states, secondary accents, charts |
| Aspire tint | `#E6F1EF` | Panels, soft callouts, table stripes |
| Aspire gold | `#BE903E` | Sparse accent only |
| Aspire ink | `#292C34` | Body text instead of pure black |
| Aspire neutral | `#F2F2F2` | Light app background |

## Typography

- Headings: Work Sans, semibold where available.
- Body: Open Sans where available.
- Fallbacks: system UI fonts are acceptable for offline/internal use.

## Rules For Future UI Work

- Use approved Aspire logo files as-is. Do not recolour, stretch, crop, distort, or rebuild the logo in text.
- Show Starman version and date on released internal tools.
- Keep gold sparse. Use it for small accents, not large fills or body text.
- Keep Starman's Advisor OS interface professional and restrained: dense operational screens, clear hierarchy, no decorative clutter.
- Do not make brand claims that imply production approval, real-data readiness, or compliance sign-off before the pilot gates are complete.
